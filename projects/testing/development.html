Portfolio

Context:

My portfolio page is a both a showcase of my learning outcomes and my development skills. The page itself is meant to showcase fun things I’ve been learning, incorporated into the design itself. I like developing fun projects, so for me it was important to incorporate this into my site.

Version control:

From the beginning I’ve found it important to keep a Github repo where changes are kept and so I can always have a backup when something breaks. I keep this regularly updated.

Development: My initial portfolio design originated from learning and playing around with Javascript, challenging myself to make interesting concepts. After I've received feedback to take a few steps back and think about how my website should look and how it should function. Inspired by Star Trek's LCARS interface, I've created a design that to me is both visually appealing and functional. JSON loader: An important part for ease of use in developing my website is making a JSON based loader. Instead of having to manually add projects to the website, I can now just add a new project to the JSON file and it will automatically be added to the website, including the appropriate images and links. The loader also gives the URL an identifier so that the page can be bookmarked or remembered without having to reload the JSON, as it isn't loading in a seperate HTML page. GSAP and animation: I had no experience with GSAP but I've wanted to make my site animated for a long time, which you can see the result of on this same page. GSAP is a great easy to use library that lets me animate my page without having to use keyframes (as often). Easy to understand code: Something which is important to me is that my code is easy to understand to both me and other people, that's why I always add comments to my code and use 1TBS indentation. Seen on the Github. Reflection: I enjoy developing fun things, I hope I’ll be able to get more opportunities for development. One thing I had issues with was the JSON loader. Making it from scratch was a nice challenge and I had thought this would make it more manageable to add in new content. However, this proved to not be the case. Adding in images inside text proved to be too much of a hassle and the entire website would look the same. Text would always be in roughly the same place, with no breaks in between. That’s why I had to make the tough decision to go back to regular HTML and CSS.
