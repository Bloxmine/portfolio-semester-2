From the feedback we received during the user test we did with Penny [LINK TO LO1] I’ve set up plans in iterating on the site:

Iteration 1:Focus on Click Reduction

Context: We conducted user testing with Penny, who had previously tested the original website. This allowed us to compare user experience across iterations. Findings: Penny's initial interaction revealed a key issue: the information architecture required too many clicks to find relevant content, particularly testimonials from ex-students. These were located on the main page but lacked a clear title, leading to difficulty in discovery.

Action: Based on this feedback, we prioritized information architecture in the redesign. We restructured the content to minimize clicks needed to find key information. For example, testimonials were given a prominent section with a clear title on the homepage.

Iteration 2: Refining the Easter Egg

Context: While Penny appreciated the easter egg (Konami code interaction), she and other testers found it challenging to discover. Their attempts involved random clicks around the website, indicating a lack of intuitive discovery methods. Findings: The easter egg's obscurity, while intended to be subtle, hindered its enjoyment for a broader audience.

Action: We acknowledged the value of the easter egg for the target audience but recognized the need for refinement. Future iterations will explore ways to make the easter egg more discoverable, potentially through subtle hints or alternative triggers, while maintaining a clean user experience.

Thanks to the user test, I came up with an idea to add a small easter egg when pressing the Fontys logo on the top left of the page. This is one of the first areas which gets clicked on so to me it made sense to add a nod to the Konami code. Now the player character from Contra walks past. Contra was the game the Konami code originated from.

Portfolio

Context:

My portfolio has seen many different iterations. Starting from learning how to use Javascript better and making a few experiments for my site design, then receiving feedback and starting again but this time instead of beginning with programming an developing the design from there, I collected inspiration.

Professional design tools:

For my document I’ve used Affinity Publisher to create a custom layout and design. This allows me to do much more than I would’ve been able in Word. One part of the document is more of a mood board while other parts are more focused on text, but the text is able to be placed anywhere.

Iteratively:

Design: The design of the portfolio has seen many differences. I originally wanted to use a more 2000’s Flash inspired website. But I received harsh feedback about the design. So I started thinking and iterating on new designs, which I’ve showcased in a design document. The site radically changed designs and from there I kept building on it until I was satisfied about the looks. However, the way the content was presented changed even more. From the feedback I received I changed how the learning outcomes are presented:

- First version had every project I’ve worked on on one single page, the user could select an outcome and the projects relevant would be highlighted. This caused issues when having to add more explanation as multiple outcomes could be selected at the same time.

- Second version changed the way learining outcomes were display into categories. Every one of the five has a different tab. Once clicked you’ll see every project and explanations about how the learning outcome was explored and used.

Programming: How I developed my website changed quite a bit. Based on the feedback received during the first portfolio review, I had scrap the current site and start over again and this time first designing and researching before actually coding. This resulted in a better manageable website but I also discussed with a few teachers whether or not to use some sort of JSON loading system. I decided on using it because it seemed better with large amounts of text, not having to touch code. However, I eventually found it to just be too much work once I wanted to change up the layout and add in images and galleries into the pages. So I went back to ‘hardcoding’ the page content into the HTML. This gave me a much better result but it did take some extra effort. I did learn how a JSON loader works and now have a better understanding of JSON file structure itself. So it wasn’t all for nothing.
